# Picture Day Asset Generator

A browser-based tool for PhotoDay studios to generate branded marketing assets — no design software or third-party accounts required. Assets are rendered instantly in-browser using HTML Canvas and exported as PNG files ready for email, web, print, and social.

## What It Generates

| Asset | Dimensions | Use |
|-------|-----------|-----|
| Email / Flyer Banner | 1200 × 545 px | Email headers, printed flyers |
| Vertical Web Banner | 300 × 600 px | Sidebar ads, website placements |
| Horizontal Web Banner | 728 × 90 px | Leaderboard ads, email footers |
| 4:5 Social Graphic | 1080 × 1350 px | Instagram, Facebook posts |
| Reminder Card | 3.5" × 5" @ 150 dpi | Printed handout cards |
| Picture Day Poster | 20" × 30" @ 72 dpi | Studio / school lobby posters |

All assets are fully branded with the studio's:
- Primary & secondary colors
- Logo (loaded from URL)
- Job name, picture day date, gallery type, and AdvancePay CTA

---

## How It Works

1. Open the tool in any modern browser
2. Fill out **4 steps**: Job Details → Brand Colors → Logo → Select Assets
3. Click **Generate** — assets render instantly on-screen using HTML Canvas
4. Download each PNG directly — no login, no upload, no Canva needed

## Branching & Deploy Workflow

This repo uses a **3-tier branch strategy** designed for human-gated production deploys:

```
dev  →  (AI codes here)
 ↓
test →  (Human QA)
 ↓
main →  (Human approves deploy → production)
```

### Branch rules

| Branch | Who commits | Deploys to | Protected |
|--------|------------|------------|-----------|
| `dev` | AI / engineers | Vercel preview | No |
| `test` | Promoted from `dev` | Vercel preview (stable QA URL) | Recommended |
| `main` | Promoted from `test` (human approval only) | Vercel production | **Yes** |

### Workflow

**Iteration cycle:**
```bash
# 1. AI makes changes on dev
git checkout dev
# ... make changes ...
git push origin dev

# 2. When ready for QA, open PR: dev → test
# Human reviews and merges

# 3. After QA passes, open PR: test → main
# Human approves and merges → auto-deploys to production
```

## Folding Into the PhotoDay App

This tool is built as a standalone static HTML file with **zero external dependencies** — all rendering runs in the browser via Canvas API. Integration assumptions:

- **Frontend framework**: Can be wrapped as a React component; Canvas renderer functions are modular and portable
- **Auth**: Currently unprotected — add PhotoDay session token check at the wrapper level
- **Routing**: Mount at `/tools/asset-generator` or as a modal/drawer in the job flow
- **Logo loading**: Currently fetched via user-provided URL; could pull from the job's PhotoDay CDN URL
- **Colors**: Could auto-populate from the studio's brand settings in PhotoDay
- **Output**: PNG data URLs — can be posted directly to PhotoDay's media API instead of downloaded

## Scripts

### `scripts/generate_kit_pdf.py`

Generates a branded "Picture Day Kit" PDF using [ReportLab](https://www.reportlab.com/).

**Requirements:**
```bash
pip install reportlab pillow requests
```

**Usage:**
```bash
python3 scripts/generate_kit_pdf.py config.json output.pdf
```

The config JSON is generated by the HTML tool and passed to this script for PDF assembly.

---

## Tech Stack

- **Frontend**: Vanilla HTML/CSS/JS — no build step, no dependencies
- **Canvas rendering**: Native browser `<canvas>` API
- **PDF generation**: Python + ReportLab (local/server-side, optional)
- **Hosting**: Vercel (static)
